混合专家大脑的杠杆点不在装什么软件，在"一个真相源 + 四器官不重叠 + 三式混合检索的查问反射 + 代谢循环"这套刻进 SOP 的活体器官系统——工具是仆人，结构是主人。
S知识工作者笔记越攒越多，AI 越搜越糊，token 越烧越多——"第二大脑"在规模化时反而宕机。 C纯 wiki+index 在规模化后精度数学性下降；纯 RAG 没编译骨架；纯云 AI 不懂你的私域。三者各有死穴、缺一不可。 Q怎么构建一套既能精确又能语义、既本地又能扩展、既懂你又能跨项目复用的"混合专家大脑"？ A五系统活体代谢架构：LLM-Wiki 真相源 × QMD 混合检索 × GitHub 脊柱 × NotebookLM 深研 × 协作运营窗口，靠"代谢 > 囤积"招牌命题、五器官铁律和三方伦理红线驱动复利飞轮。

核心议题树（MECE）· 5 个互斥穷尽的子问题

① 真相源 · Source of Truth

本地优先 + 不可变 raw + AI 友好 markdown，才是知识能复利的地基。
  • 三层架构：raw 不可变 / wiki 可演化 / insights 人独占
  • Schema 是第四层：CLAUDE.md 让任何 AI 1500 字内定位
  • 单一真相源铁律：多源 = 没源
So-What：换编辑器/App 都能延续，因为底层是纯文本 + schema 不是 SaaS 锁定。
Action：先建三层 + Claude.md，喂前 10 篇验证自动长出概念页。
🟢 已 N=1 验证

② 检索器官 · Retrieval Organ

BM25 抓字、向量抓意、混合 + 重排默认、Structured Query 是杀手锏——4 模式信息量差 10×。
  • 编译 vs 检索互补铁律：wiki 管骨架，RAG 管 raw 细节
  • 中文必切 Qwen3：默认英文模型质量差一截
  • 实测省 96% token：15k → 500（Andrew Levine vault）
So-What：把"AI 帮我查"从 token 黑洞变成秒级精准片段。
Action：装 QMD + 切 Qwen3 + 全量索引 + 冒烟验真。
🟢 已 N=1 验证

③ 治理纪律 · Discipline

六块肌肉 + 三方红线——工具替不了的"人的本事"决定大脑成败。
  • 六肌肉：站位/摄入/检索/验真/复用/复利 + 自免疫
  • 三红线：真相不污染 × AI 5 类判断不替 × 数据主体 PII
  • 头号红线：AI 自报告失实——负证据换工具复验
So-What：能力不长，工具堆再多也只是数字仓鼠。
Action：每个工具问"它服务我哪块肌肉"；答不上就别装。
🟡 持续训练

④ 成长路径 · Growth Path

B2 通感就够 95% 个人用户——B3/B4 是按需，不是必经。
  • B1 立脑(0-1周)：真相源能问答
  • B2 通感(1-2周)：QMD 检索 + GitHub 备份 ← 多数人合格止损点
  • B3 长器官(1-2月按需)：NotebookLM / 协作窗口
  • B4 活体(持续)：cron 自代谢 + 反哺循环
So-What：跳阶必崩；停在合格的阶段不是失败是止损。
Action：用阶段诊断卡定位"我在 B 几、卡在哪、下一步做什么"。
🟢 N=1 走完到 B4

⑤ 复利飞轮 · Compounding

代谢 > 囤积——价值由循环跑得多顺决定，不是存了多少。
  • 五环循环：摄入→消化→检索→产出→反哺
  • 反哺让大脑越用越懂你：喂你的思考 → AI 学你的判断
  • 复用收敛 vs 重写发散：知识必须聚焦
So-What：图书馆静态，越大越烂；活体动态，越大越聪明。
Action：每月反哺一次——产出效果回标知识页，让飞轮开始转。

失真自报 · 这套蓝图的边界与未验证假设（显式承认）

  • N=1 验证：完整 B1→B4 走通 N=1（单一用户实操部署），五系统全器官 N=1；个人用户 B2 止损点是经验外推，未做用户研究
  • 中文场景偏置：Qwen3 + 国内 HF 镜像是中文用户的最优解；英文 vault 应切回默认 embeddinggemma，本蓝图未充分覆盖
  • "代谢 > 囤积"招牌命题：哲学层面成立，工程层面的"什么算代谢成功"缺定量指标（feedback_score 是 N=1 探索）
  • 器官分工铁律：当工具升级时（如 GitHub 增加 Copilot 检索）可能模糊器官边界，铁律需 v2 复审
  • 三方红线伦理：标 pending-review；适用法域/合规细节需各地具体咨询
  • 反例边界：5 类反例（精确词稀释/私有术语/hyde 写偏/小库/跨语言）来自实操，非穷举——可能存在未识别的失败模式
