# 🎯 混合检索经典案例

> 同一个意图，**四种问法跑下来差别是什么** —— 一次性看穿 QMD 三式 + Structured Query 的精髓。
> 配套绿皮书卷三第 14 章（混合检索三式的原理）。

## 0. 场景设定

**你要写一篇关于"深度工作"的文章**，需要从自己库里挖：
- 一段精确论点（含数字/术语）做引用
- 一段意思像但说法不同的洞察（可能埋在 raw 里）
- 关于这主题的全面相关材料
- 一个最深、最有冲击力的切入角度

**同一个意图 → 四种问法 → 看本质差别。**

---

## 1. 四种问法 · 同主题对比

### 模式 ① BM25 精确（`qmd search`）
```bash
qmd search '"深度工作" Cal Newport 4 小时' -c brain
```
- **返回**：标题/正文**字面命中**"深度工作"+"Cal Newport"+"4 小时"的文件
- **强项**：找你记得**确切词**的引用（版本号/专名/特殊数字）
- **死穴**：你想找的内容若用了同义词（"专注力"/"心流"）→ **零命中**

### 模式 ② 向量语义（`qmd vsearch`）
```bash
qmd vsearch "我对碎片化注意力的反思" -c brain
```
- **返回**：意思相近但字面不同的内容——可能命中 raw 里一段"刷手机的代价"、"短视频毒性"，**一字没重复**
- **强项**：找**意思像字面不像**的零散观点，尤其 raw 原文细节
- **死穴**：搜"v2.1.0"会给你 v2.0/v1.9（语义太像但版本不对）

### 模式 ③ 混合 + 重排（`qmd query`）—— 默认推荐
```bash
qmd query "深度工作怎么实践 注意力切换的代价" -c brain
```
- **底层流程**：query expansion(小模型扩成多变体) → 每变体并行跑 BM25+向量 → RRF 合并 → qwen3-reranker 重排 → 按位置加权出最相关 3-5 段
- **强项**：90% 日常查询的最优解；既不漏精确词，也不漏语义近邻
- **死穴**：当你**确切知道**要什么时反而比 ①/② 慢一点

### 模式 ④ Structured Query（杀手锏）
```bash
qmd query $'intent: 找深度工作的反共识观点+真实代价数据,不要泛泛方法论\nlex: 深度工作 注意力切换代价 心流 task-switching cost\nvec: 现代人为什么难以进入深度工作 多任务的隐性损耗\nhyde: 一段笔记指出,平均每次切换任务后大脑需要 23 分钟才能完全恢复深度状态;现代知识工作者每天被打断 50+ 次,实际深度时间不足 2 小时。这是注意力经济的隐性税。' -c brain
```
- **四段信号**：
  - `intent:` 排除"我不想要的"（不要泛泛方法论）
  - `lex:` 给精确锚定词
  - `vec:` 给自然语言描述
  - `hyde:` **假装答案**——这是被低估的杀手——把"我心目中理想的命中页长什么样"写出来，让模型按这个去匹配
- **强项**：**前 3 模式都漏的源，hyde 段能挖出来**——因为它直接告诉模型"理想答案长什么样"，跳过了"用户问题→答案"这一跳的歧义
- **死穴**：写起来重，只用在最重要的查询上（写文章/做决策/最深挖那一次）

## 2. 第一性 · 四种检索的本质差别

| 模式 | 哲学 | 什么时候选它 |
|---|---|---|
| BM25 | "你写什么字我搜什么字" | 知道确切版本号/专名/代码符号 |
| 向量 | "你想什么意思我猜什么意思" | 完全口语化、不知关键词、找跨类细节 |
| 混合 + 重排 | "两路同时来，我挑最准的" | **大多数日常查询的默认** |
| Structured | "你给我多维信号，我极致挖掘" | 写文章/做决策/最重要的那次问 |

**核心洞察**：源文章里 Tobi 引用的混合检索流程，本质是 Google 搜索和 Anthropic 内部检索同一套路——**query expansion → 并行 BM25+向量 → RRF 合并 → reranker 重排**。QMD 把它打包了，你只管问，工程细节它处理。

## 3. 决策树（日常该走哪条）

```
你要查的东西…
│
├─ 知道确切词/版本号/专名 ───────→ qmd search          (1 秒，字面命中)
├─ 完全口语化/记不清关键词 ─────→ qmd vsearch         (向量猜意思)
├─ 大多数情况 ───────────────→ qmd query           (混合+重排，默认)
└─ 写文章/做决策/最重要的那次问 → qmd query + Structured (intent/lex/vec/hyde)
```

## 4. 可复用模板（直接抄改）

### 模板 A · 找原话/raw 细节（场景三）
```bash
qmd query "那句话的大意 + 大概语境" -c brain
```

### 模板 B · 找历史决策的"为什么"
```bash
qmd query $'intent: 找历史决策的理由,不要泛泛的方法论\nlex: 决策 取舍 风险\nvec: 当时为什么这么定 关键考量\nhyde: 经过 X×Y×Z 三方面权衡,选择 A 而非 B,因为...' -c brain
```

### 模板 C · 验真/找精确数字
```bash
qmd search "确切数字或专名" -c brain    # 先精确
qmd query "上下文" -c brain            # 再核语境
```

### 模板 D · 写文章前找素材（极致挖掘）
```bash
qmd query $'intent: 找关于 [主题] 的反共识观点+真实数据,不要泛泛方法论\nlex: [3-5 个最强锚定词]\nvec: [一句话用户原话描述]\nhyde: [50-100 字假装答案段,带核心术语+案例+数字]' -c brain -n 10
```

## 5. 训练阶梯（把案例变肌肉）

| 时间 | 训练动作 | 自检 |
|---|---|---|
| 第 1 周 | 同一个真实查询，4 种模式各跑一遍，肉眼对比差别 | 能说出每种模式漏了什么、抓住了什么 |
| 第 1 月 | 默认改用 `qmd query`，纠结时再切 search/vsearch | 检索命中率明显提升 |
| 第 3 月 | 重要查询学会写 Structured Query（重点练 hyde 段） | 写文章/做决策前的查询用它 |
| 第 6 月 | 检索成本能反射，token 大降，召回率显著提升 | 同问题比半年前省 ≥80% token |

## 6. 一句话记忆

> **BM25 抓字、向量抓意、混合+重排默认、Structured 杀手锏。**
> **Hyde（假装答案）是被低估的招——前 3 模式都漏的源，它能挖出来。**
