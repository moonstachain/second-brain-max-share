## A1 · 五系统角色速查卡

> 一句话：**一个真相源 + 四器官，不重叠。**（详见卷二第 8 章）

| 系统 | 器官 | 唯一职责 | 绝不做 | 何时用 |
|---|---|---|---|---|
| 🧠 OB 知识库 | 大脑本体 | 真相源（原料+知识+洞察） | 不做全文语义检索 | 一切知识的家 |
| 🔍 QMD | 检索器官 | 全 vault 混合检索 | 不存知识/不当真相源 | 找东西（自动走） |
| 🦴 GitHub | 脊柱 | 备份/版本/跨机/技能 | 不检索/不当真相源 | 备份（无感） |
| 🔬 NotebookLM | 深研器官 | 多源深综+音频 | 不做日常检索 | 深研/出播客 |
| 🪟 飞书 | 运营窗口 | 看板/协作/PII业务 | 不当真相源 | 给团队/移动端看 |

**人机边界（5 类 AI 永不替代，卷七第 36 章）**：价值排序 / 不可逆决策 / 伦理判断 / 品味判断 / 意义赋予。

## A2 · 三条治理线 + 编译vs检索 速查卡

> 详见卷二第 6、9 章

**三条治理线（最高律，不可破）**：
1. **单一真相源**——知识库 说了算，器官冲突以它为准
2. **Layer1 只读不可变**——原料 AI 只读，知识写 Layer2
3. **本地优先**——核心本地跑，云挂了大脑还活

**六条架构铁律（卷二全卷）**：
| # | 铁律 | 一句话 |
|---|---|---|
| 1 | 单一真相源 | 多真相源=没真相源 |
| 2 | Layer1 只读 | 不改原文，溯源可信 |
| 3 | 五器官不重叠 | 每个只干一件事 |
| 4 | 编译vs检索互补 | wiki 管骨架，QMD 管细节 |
| 5 | facts-raw 分层 | 纪要数字默认存疑 |
| 6 | PII 物理隔离 | 本地可问答，git 不带走 |

**编译 vs 检索（卷二第 9 章）**：
- 编译层（wiki+index）= 把已知重要的编译成结构化骨架
- 检索层（QMD）= 在全 vault 找跟 query 相关的零散片段（含未编译 raw）
- **互补，不替代**

## A3 · 场景 × 系统 × 调用法 交叉表

> 详见卷五第 28 章

| 你想做 | 走哪个 | 怎么调 |
|---|---|---|
| 找记不清出处的细节/原话 | QMD | `qmd query "意思" -c wiki` |
| 找确切版本号/专名 | QMD(BM25) | `qmd search "词" -c wiki` |
| 找模糊概念（意思像字面不像） | QMD(向量) | `qmd vsearch "意思" -c wiki` |
| 读某概念的结构化知识 | wiki | 打开 concept 页 |
| 任务前拉背景 | knowledge-运营者 | `/knowledge-运营者 "主题"` |
| 对一摞源深度综合/音频 | NotebookLM | 选库深研 |
| 出内容（公众号/视频/星球） | content_factory | 跑产出管道 |
| 给团队/移动端看 | 飞书 | bridge 同步的看板 |
| 备份/换机器 | GitHub | 自动 push（无感） |

**默认法则**：拿不准就用 `qmd query`（自动混合+重排，最省心）。

## A5 · QMD 三式检索 + 命令速查卡

> 详见卷三第 14 章 + 卷四第 20 章

**三式检索**：
```bash
qmd query "完整问题" -c wiki      # 推荐:混合(BM25+向量+重排),最准
qmd vsearch "模糊意思" -c wiki    # 纯向量:意思像字面不像
qmd search "精确词" -c wiki       # 纯BM25:版本号/专名/确切词
# 不加 -c 跨所有 collection 一起搜
```

**维护命令**：
```bash
qmd update              # 增量索引(新笔记后随手跑,秒级)
qmd status              # 看索引健康/向量数/模型
qmd embed -f -c wiki    # 仅换 embedding 模型时全量重建
qmd collection add <路径> --name <名>  # 加新知识库
```

**中文知识库三必做**：
1. `QMD_EMBED_MODEL` / config 切 `Qwen/Qwen3-Embedding-0.6B`
2. 国内设 `HF_ENDPOINT=https://hf-mirror.com`
3. 改模型必须改 `~/.config/qmd/index.yml`（config > 环境变量）+ `embed -f` 重建 + 重启 Claude Code

## A6 · B1-B4 阶段诊断卡

> 详见卷六第 35 章

```
你的大脑现在能做什么？
│
├ 还不能基于我的知识回答 ──────→ B1前/B1  → 卷六第31章(立脑)
├ 能答但搜不准/raw细节找不回 ──→ 卡B1→B2 → 卷六第32章(接QMD+Qwen3)
├ 检索准了但怕丢/换机器麻烦 ───→ B2     → 补GitHub(第32章)
├ 能用且不丢,但要深研/给团队看 ─→ 按需B3 → 卷六第33章
├ 器官齐了但全靠手动养会断 ────→ B4     → 卷六第34章(自动化)
└ 自运转了但出过覆盖/污染事故 ──→ B4加固 → 第34章(merge/验证)
```

**四阶完成标准**：
| 阶 | 完成标准 | 周期 |
|---|---|---|
| B1 立脑 | 真相源能摄入消化能问答 | 0-1 周 |
| B2 通感 | 检索冒烟通过+可恢复 | 1-2 周 |
| B3 长器官 | 按需接深研/运营+产物回流 | 1-2 月（可选） |
| B4 活体 | 自代谢+反哺循环（无终点） | 持续（可选） |

**关键认知**：停在哪阶都合格。个人 B2 够用，业务才 B3，自运转才 B4。

## A7 · 真实教训档案（带血的坑）

> 详见各卷案例 · 全部来自参考实现真实翻车

| 坑 | 症状 | 解法 | 出处 |
|---|---|---|---|
| **QMD 装错目标** | 检索接到工作目录而非真大脑 | 先认准真相源再接检索 | 卷一第5章 |
| **HF 下载卡死** | 直连 HuggingFace 卡"Gathering information"零下载 | `HF_ENDPOINT=https://hf-mirror.com` | 卷四第20章 |
| **config 优先级** | 设了环境变量但 QMD 用旧模型 | 改 `~/.config/qmd/index.yml`，MCP 不读 shell env | 卷四第20章 |
| **维度不匹配** | 换模型后 MCP 报 768 vs 1024 | `embed -f` 重建 + 重启 Claude Code | 卷二第8章 |
| **纪要数字放大~4倍** | 转写财务数字夸大 | facts 层校正，raw 默认存疑 | 卷二第10章 |
| **自报告失实** | AI说"装好了"实则没有 | 负证据换工具复验 | 卷三第15章 |
| **便利样本偏置** | n=8 得88%，随机n=38真值31% | 固定seed随机抽n≥30报来源 | 卷三第17章 |
| **持久化覆盖(49→2)** | 空数据fallback覆盖完整版 | merge不overwrite，推前fetch | 卷二第6章 |
| **改名挂 git** | 主机名变致 user.email 非法，全机提交挂 | `git config --global user.email/name` | 卷四第21章 |
| **launchd TCC 拦截** | `~/Documents/`下脚本被隐私保护拦 | bash加完整磁盘访问/搬目录/迁GitHub Actions | 卷四第24章 |
| **Obsidian 看不到新文件** | 外部建的文件树不刷新 | Cmd+R重载/重启；搜文件名非文件夹名 | 卷四第19章 |

**母题**：**自报告失实是 AI 协作头号红线**——"它说成功"≠"真成功"，唯一防线是你的验真肌肉（卷三第15章）。

## A8 · 复盘模板（日/周/月代谢复盘）

> 详见卷五第 29 章 · Mode A 维护节奏

**日复盘（随手，1 分钟）**：
- [ ] 今天看到的好东西，进 raw 了吗？（不囤收藏夹）
- [ ] 新笔记 `qmd update` 了吗？

**周复盘（每周，10 分钟）**：
- [ ] 跑 wiki-lint（L1-L8）：断链/孤页/矛盾几个？修了吗？
- [ ] 本周 raw 都消化了吗？有积压吗？
- [ ] 检索调用顺手吗？有没有用错器官？

**月复盘（每月，30 分钟）**：
- [ ] 看代谢仪表盘：知识页/向量数在长还是停滞？
- [ ] 阶段诊断（A6）：我在 B几？这月进阶了吗？还是停滞？
- [ ] 复利检查：AI 产出有没有更懂我的风格？（卷三第18章）
- [ ] 喂了几篇自己的思考？（最高价值养料，卷三第13章）
- [ ] 三方红线（A1/卷七）有没有破？

**复盘心法**：复盘不是记流水账，是诊断大脑在"越用越强"还是"越用越烂"——前者继续，后者找卡点（A6/A7）。

## A9 · 一键部署包速查卡

> 配套可执行脚本在 `部署包/`，把 A7 教训档案的坑全预修进去了。详见 `部署包/README-如何一键部署.md`

**最快·真·一键**：
```bash
bash 部署包/一键部署.sh ~/你的vault brain   # 串起 step0→1→(step2暂停粘提示词)→3→4
```

**分步五步流程**（想手动控制时）：
```bash
① bash 部署包/step0-环境自检.sh                      # 查依赖，缺啥补啥
② bash 部署包/step1-装QMD检索器官.sh                 # 装qmd+plugin+配国内HF镜像
③ 复制 部署包/CLAUDE.md.模板 → 你的vault/CLAUDE.md
   按 部署包/step2-建脑主提示词.md 粘给 Claude Code   # 建大脑本体(B1立脑)
④ bash 部署包/step3-接检索并索引.sh ~/你的vault brain # 接检索+Qwen3+索引(B2通感)
⑤ bash 部署包/step4-验真冒烟.sh brain "中文测试词"     # 冒烟验真,全绿即成功
# 成功后日常用：qmd query "你的问题" -c brain
```

**为什么不是"一个脚本全自动"**（绿皮书"不过度承诺"精神）：Obsidian 是 GUI（要你点开）、建知识库结构是 AI 干的活（step2 给主提示词）。这套是"脚本做确定性 + 提示词做 AI 部分 + 验真兜底"的最高自动化形态。

**脚本已预修的坑**：装错目标（强制传 vault 绝对路径）/ HF 下载卡（自动配镜像）/ config 优先级（直接改 index.yml 写 Qwen3，不靠环境变量）/ 中文质量（默认 Qwen3）/ 装好≠能用（step4 强制冒烟）。
