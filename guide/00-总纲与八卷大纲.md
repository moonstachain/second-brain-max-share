# 本地专家大脑绿皮书
## 五系统活体知识代谢 · AI 原生复刻指南

> **一句话**：教你从 0 复刻一套"用得越久越懂你、越大越不卡"的本地专家大脑——
> 一个真相源（OB 知识库）+ 四个器官（QMD 检索 / GitHub 脊柱 / NotebookLM 深研 / 飞书运营）+ 一套活体代谢循环。
>
> **本书状态**：`v0.1 完整深草稿`（8 卷全文 + 一键部署包 + 概念图；绿皮书需 4-8 轮迭代到 v1.0）

## 0. 这本书是什么 / 不是什么

| 是 | 不是 |
|---|---|
| 从第一性原理讲清"为什么第二大脑会宕机、怎么救" | 不是"装 QMD 教程"（那只是卷四一节） |
| 五系统**整体架构** + 部署搭建 + 使用攻略 + 运维进化的全链路 | 不是单点工具说明书 |
| 让生态里的人**从 0 复刻一套自己的** | 不是任何人的私有业务内容（PII / 凭证不入书） |
| 8 卷层层递进，每章挂回真实概念页，零捏造 | 不是通用知识管理鸡汤 |

**参考实现**：五系统第二大脑（架构定稿见 本书架构），底层第一性来自 LLM Wiki 模式（Karpathy）+ 源文章《你的第二大脑需要 QMD 才能健康扩展》。

**读者画像**：你的团队/生态外部可复制者，中级技术素养（会命令行 / Claude Code，有 Obsidian 或愿意学），当下痛点 = 笔记越攒 AI 越搜不准、token 越烧越多、知识成了"数字仓鼠囤积"用不起来。

## 1. 招牌命题（全书脊柱概念）

> **「活体知识代谢」 Metabolism over Hoarding**
>
> 第二大脑的本质不是"存得多"（图书馆 / 仓库），而是"代谢得动"（活体器官系统）。
> 价值 = 摄入 → 消化 → 检索 → 产出 → 反哺 这条**活体循环**跑得有多顺，
> 而不是你囤了多少 G 的笔记。
>
> 丢掉这个概念，本书就退化成一篇装软件教程。它在**卷一第 3 章**首次命名，
> 并驱动卷四（器官工具）、卷五（代谢节奏）、卷六（成长阶段）的结构。

衍生的第一性二分（贯穿全书）：
- **编译知识 vs 全文检索**：wiki+index 管"已编译的骨架知识"，RAG/QMD 管"全 vault 的零散细节" —— 互补，不替代。
- **单一真相源 × 多器官辐射**：一个大脑本体说了算，四个器官各司其职不重叠。

## 2. 八卷总览（层层递进）

```
卷一 · 为什么(Why)     第二大脑为什么会宕机 —— 活体代谢世界观
   ↓ 读者内化"为什么不能只囤积"
卷二 · 是什么(What)    五器官架构铁律 —— 不可破的宪法
   ↓ 读者知道"哪些规则不能破"
卷三 · 怎么练(How)     运营活体大脑的肌肉 —— 元能力
   ↓ 读者知道"自己要长哪些本事"
卷四 · 用什么(Which)   五系统工具栈 —— 元工具(⚠️防工具拜物教)
   ↓ 读者知道"具体装什么"
卷五 · 何时做(When)    部署顺序 + 日常代谢节奏 —— 元节奏
   ↓ 读者知道"按什么顺序搭、搭完怎么养"
卷六 · 怎么长(Path)    B1立脑→B2通感→B3长器官→B4活体 —— 元成长
   ↓ 读者知道"大脑怎么逐阶段长大"
卷七 · 不可为(Ethics)  真相/AI/数据主体 三方红线 —— 元伦理
   ↓ 读者知道"什么绝对不能做"
卷八 · 速查(Reference) A1-A8 桌面速查卡 —— 元附录
```

每卷之间的"递进门禁"：没有世界观（卷一），架构铁律（卷二）会显得武断；没有铁律，能力训练（卷三）没靶子；没有能力靶子，工具（卷四）会被拜物；没有工具，节奏（卷五）落空；没有节奏，成长路径（卷六）排不出序；没有路径，伦理（卷七）变成空洞道德；没有伦理，速查卡（卷八）没有治理。

## 3. 逐卷大纲（核心问题 + 核心主张）
