# 本地专家大脑 · 一键部署包

> 配套《本地专家大脑绿皮书》。目标：**你拿到这套，照着跑就能部署成功。**
> 设计原则：把绿皮书 A7 教训档案里的坑**全部预修进脚本**，让你不必再踩一遍。

## 最快：真·一键编排器

```bash
bash 部署包/一键部署.sh ~/你的vault brain
```

`一键部署.sh` 把 step0→1→**(step2 暂停等你粘提示词)**→3→4 串成一条线跑完，每步有横幅+失败即停。中途到建大脑本体时会**自动复制 CLAUDE.md 模板、打印主提示词、暂停**，等你粘进 Claude Code 建好回车继续。不想一步步敲命令就用它。

下面是分步版（想手动控制每一步、或某步单独重跑时用）：

## 这套包做什么

| 文件 | 作用 | 类型 |
|---|---|---|
| `一键部署.sh` | **总编排**：串起下面 step0/1/3/4，step2 交互暂停 | 脚本（编排） |
| `step0-环境自检.sh` | 检查 node/npm/claude/git/obsidian 是否就位，缺啥给安装命令 | 脚本（只读） |
| `step1-装QMD检索器官.sh` | 装 qmd CLI+plugin，配国内 HF 镜像 | 脚本（幂等） |
| `CLAUDE.md.模板` | 大脑本体的宪法文件（三层架构规则） | 模板 |
| `step2-建脑主提示词.md` | 粘给 Claude Code，让它建好 vault 骨架 | 提示词 |
| `step3-接检索并索引.sh` | 接 QMD 到真相源，切中文 Qwen3，全量索引 | 脚本（幂等） |
| `step4-验真冒烟.sh` | 验证检索真能召回（装好≠能用） | 脚本（只读） |

## 为什么不是"一个脚本全自动"

诚实说（这正是绿皮书"不过度承诺/验真"精神）：**两件事无法纯脚本自动化**——
1. Obsidian 是 GUI 应用（`brew install --cask obsidian` 能装，但要你点开）
2. "建知识库结构"是 AI 干的活（step2 给你主提示词，让 Claude Code 替你建）

所以这套是 **"脚本做确定性部分 + 提示词做 AI 部分 + 验真兜底"** 的最高自动化形态，而非一个骗人的"全自动黑盒"。

## 五步部署流程

```
① bash step0-环境自检.sh                    # 看缺什么，补齐
② bash step1-装QMD检索器官.sh                # 装检索器官+配镜像
③ 复制 CLAUDE.md.模板 → 你的vault/CLAUDE.md
   按 step2-建脑主提示词.md 粘给 Claude Code  # 建大脑本体(B1立脑)
   → 喂几篇材料，"帮我录入"，确认长出概念页
④ bash step3-接检索并索引.sh ~/你的vault brain # 接检索+Qwen3+索引(B2通感)
⑤ bash step4-验真冒烟.sh brain "你的中文测试词"  # 冒烟验真
```

跑完 ⑤ 全绿 = 你的本地专家大脑 B1+B2 部署成功，可日常用：
```bash
qmd query "你的问题" -c brain
```

## 已为你预修的坑（A7 教训档案）

| 坑 | 这套包怎么预修 |
|---|---|
| 装错目标 | step3 强制传 vault 绝对路径，传错目录直接报错 |
| HF 下载卡死 | step1 自动把 HF_ENDPOINT 镜像写进 shell rc |
| config 优先级（环境变量被无视） | step3 直接改 `~/.config/qmd/index.yml` 写 Qwen3，不靠环境变量 |
| 中文检索质量差 | step3 默认切 Qwen3-Embedding（中文必做） |
| git 提交全挂 | step0 检查 git 全局身份并提示配置 |
| 装好≠能用 | step4 强制冒烟验真（向量数>0 + Qwen3生效 + 中文真召回） |
| 维度不匹配 | step3 末尾提示换模型后重启 Claude Code |
